# 🏢 ACME - Academic Center for Managed Engineering

Bem-vindo à organização **ACME**. Este espaço é dedicado ao desenvolvimento de soluções tecnológicas experimentais e laboratoriais, funcionando como o hub central para o ciclo de vida de software dentro do contexto acadêmico.

## 🚀 Sobre a Organização
A ACME (Academic Center for Managed Engineering) simula um ambiente corporativo real para o desenvolvimento de software escalável. Nosso foco é a convergência entre teoria acadêmica e práticas de mercado, utilizando o **GitHub** como a espinha dorsal para colaboração, versionamento e automação.

## 🎓 Contexto Acadêmico: Disciplina de Desenvolvimento de Software
Esta organização hospeda o projeto principal da disciplina de **Desenvolvimento de Software**. O objetivo é a criação de uma aplicação robusta que percorra todas as etapas do SDLC (*Software Development Life Cycle*), incluindo:

* **Planejamento:** Gestão de requisitos e tarefas via GitHub Projects.
* **Desenvolvimento:** Implementação seguindo padrões de arquitetura limpa e código manutenível.
* **Revisão:** Processo rigoroso de *Code Review* através de Pull Requests.
* **Qualidade:** Execução de testes automatizados e análise estática.
* **Deploy:** Automação de entrega contínua utilizando GitHub Actions.

## 📈 Fluxo de Trabalho (Workflow)
Para manter a organização e a qualidade do código, todos os membros devem seguir o padrão estabelecido:

1.  **Issues:** Toda nova funcionalidade ou bug deve ter uma Issue detalhada.
2.  **Branching:** Utilizamos uma variação do *GitFlow* (Feature Branches -> Develop -> Main).
3.  **Pull Requests:** Devem conter uma descrição clara das mudanças e passar por pelo menos uma aprovação de peer review.
4.  **CI/CD:** O código só é integrado após passar com sucesso por todos os workflows de validação.
